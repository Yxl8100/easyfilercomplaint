## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, checkbox/radio gap-1 |
| sm | 8px | Compact element spacing, `space-y-1.5` within form fields |
| md | 16px | Default element spacing, px-4 on inputs |
| lg | 24px | Section padding, `p-6` on summary cards |
| xl | 32px | Step content padding, `py-10` on main container |
| 2xl | 48px | Step-nav separator `pt-8 mt-8` |
| 3xl | 64px | Page-level vertical rhythm (not used in wizard; reserved for future) |

Exceptions:
- Form field groups: `space-y-5` (20px) — existing convention, do not change
- Input height: `py-3` (12px vertical) + border = ~44px total — meets touch target guidelines
- Upload drop zone minimum height: 80px (tap-safe on mobile)
- "Coming soon" badge on FCC option: inline, no extra spacing

Source: `src/app/file/[category]/page.tsx`, `src/components/forms/FormField.tsx`

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 14px (`text-sm`) | 400 | 1.5 (`leading-relaxed`) | Inter (`font-body`) |
| Label / hint | 9px (`text-[9px]`) | 400 | 1.2 | JetBrains Mono (`font-mono`), uppercase, tracking-[0.15em] |
| Heading (step title) | 20px (`text-xl`) | 700 | 1.3 | Fraunces (`font-serif`) |
| Display (review total) | 24px (`text-2xl` / `text-xl`) | 700 | 1.2 | Fraunces (`font-serif`) |

Scale: 4 sizes (9px, 14px, 20px, 24px). Weights: 2 (400, 700).

The Label / hint token at 9px (`text-[9px]`) covers ALL mono field labels AND hint/sub-label text. The previous 8px size is removed; use 9px uniformly for both purposes.

Note: The existing wizard uses `text-sm font-body` for all body copy and `font-mono text-[9px] tracking-[0.15em] uppercase` for all field labels. Phase 8 MUST use these exact classes for new elements. Do not introduce new font sizes.

Source: `src/components/forms/FormField.tsx`, `src/app/file/[category]/page.tsx`

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (`bg-bg`) | Page background, input backgrounds, card fill |
| Secondary (30%) | `#f5f5f3` (`bg-alt`) | Step summary blocks, info callout boxes, agency option hover fill |
| Dark surface | `#1c1c1c` (`bg-dark`) | Active CTA button fill, "Pay & File" button, progress bar completed steps |
| Primary text | `#1c1c1c` (`text`) | All heading and data values |
| Mid text | `#3a3a3a` (`text-mid`) | Body copy, descriptions |
| Muted text | `#6a6a6a` (`text-light`) | Labels, hints, placeholder text |
| Accent (10%) | `#b91c1c` (`accent`) | Required asterisk (`*`), error state borders, error heading text, "coming soon" badge background, terminal CTA button (Pay & File) |
| Accent background | `#fef2f2` (`accent-bg`) | Error state container background |
| Border default | `#e5e5e3` (`border`) | Input borders, card borders, dividers |
| Border dark | `#c5c5c3` (`border-dark`) | Focused input border substitute (uses `focus:border-bg-dark` = `#1c1c1c` in practice) |
| Destructive | `#b91c1c` = `accent` | Form submission errors only; no destructive actions in this phase |

Accent (`#b91c1c`) is reserved for:
1. Required field asterisk
2. Error state container border and heading text
3. "Coming soon" badge text on FCC option (WIZ-04)
4. The "Expand/collapse" text link in the review step
5. Terminal CTA button ("Pay & File") fill — `bg-accent text-white` — applies only to the final step's primary action after attestation is checked

Do NOT use accent on: normal interactive buttons (use `bg-dark`), normal interactive state, hover states. The terminal CTA exception (item 5) is intentional; it signals irreversibility of the payment action.

## Wizard Step Mapping (Phase 8)

The wizard restructures from the current 5 steps to a 6-step flow matching the product decisions:

| Step # | Label (ProgressBar) | Description |
|--------|---------------------|-------------|
| 0 | Complaint Type | Radio-card selector for the three complaint types |
| 1 | Business Info | Who are you filing against? (unchanged) |
| 2 | Details | What happened? + visit date + evidence upload |
| 3 | Agency | CA AG (active) + FCC (coming soon) display |
| 4 | Your Info | Filer information, California pre-selected |
| 5 | Review | Full summary + attestation + Pay & File |

Update `ProgressBar` `STEPS` array to: `['Complaint Type', 'Business', 'Details', 'Agency', 'Your Info', 'Review']`

Step 0 focal point: the three radio cards are the primary visual anchor — all other elements (heading, description, Continue button) are supporting. Card layout must be visually dominant relative to surrounding copy.

## Interaction Contracts

### Complaint Type Step (WIZ-01)
- Radio card selection: clicking anywhere on the card selects it
- One selection at a time (single complaint type per wizard session)
- Selected card: `border-bg-dark` border, `bg-bg` fill (no fill change — border only signals selection)
- Continue button disabled until a type is selected

### Visit Date (WIZ-02)
- Two `FormSelect` dropdowns (Month, Year) side by side
- Both are optional — no blocking validation on this field
- Default value: empty (no pre-selection)
- Month options: "January" through "December" with numeric values `01`–`12`
- Year options: current year (`2026`) down to 5 years prior (`2021`), descending order

### Evidence Upload (WIZ-03)
- Drop zone is a styled `<label>` wrapping a hidden `<input type="file">`
- Accepts: `.pdf`, `.png`, `.jpg`, `.jpeg`
- Max size: 5 MB — validated client-side before state update
- On valid file selected: replace prompt text with "File selected: {filename}" in `font-body text-sm text-text`
- On invalid: show error container (`bg-accent-bg border border-accent rounded-[6px] p-3`) with error copy
- Remove button: `font-mono text-[9px] uppercase text-text-light hover:text-accent` with text "Remove ×"
- The file object is held in React state and uploaded to `/api/upload-evidence` at checkout time (not immediately on select)

### Agency Step (WIZ-04)
- No interactivity — CA AG is the only active option, pre-selected
- CA AG card: full opacity, no checkbox (selection is implicit)
- FCC card: `opacity-50 cursor-not-allowed`, "Coming Soon" badge, no click handler
- Continue button always enabled on this step (no choice required)

### Filer Info State Pre-select (WIZ-05)
- The "State" `FormSelect` initializes with `value="CA"`
- The dropdown remains editable — user can change it
- No hint text is shown unless the user changes away from CA (this phase: no need to add a hint)

### Review + Attestation (WIZ-06)
- All summary fields shown: complaint type label, business name, agency name, approximate visit date (if entered), evidence filename (if uploaded), filer name, filer email
- Attestation checkbox is required — unchecked state: Continue button uses `bg-border text-text-light cursor-not-allowed` (same disabled style as other disabled Continue states)
- Checked state: Continue button uses `bg-accent text-white hover:bg-bg-dark` (terminal CTA — accent fill is intentional here per Color section item 5)
- No inline toggle of expanded complaint text in the simplified review — Phase 8 review is summary only
