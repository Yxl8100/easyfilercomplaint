## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding |
| sm | 8px | Compact element spacing, gap between label and input hint |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section padding within cards, `p-6` surfaces |
| xl | 32px | Top/bottom section breaks, `pt-8 mt-8` |
| 2xl | 48px | Page vertical padding, `py-10` or `py-12` |
| 3xl | 64px | Major page-level breathing room |

Exceptions:
- Touch targets on primary CTA button: minimum 44px height (accessibility — currently `px-8 py-3` on `text-[11px]` mono achieves ~40px; use `py-3.5` on success page CTA to hit 44px)
- Wizard step container: `max-w-3xl mx-auto px-6` — matches existing wizard layout exactly

## Typography

All sizes and weights match the existing codebase patterns exactly. Source: `src/app/file/[category]/page.tsx`, `src/components/Masthead.tsx`, `src/components/forms/StepNavigation.tsx`.

| Role | Font | Size | Weight | Line Height | Tracking | Usage |
|------|------|------|--------|-------------|----------|-------|
| Display | font-serif (Fraunces) | 28px (text-3xl ≈ 30px) | 700 (bold) | 1.2 | normal | Page title on success page ("Complaint Filed") |
| Heading | font-serif (Fraunces) | 16px (text-base) | 700 (bold) | 1.3 | normal | Card headings, receipt ID label, agency name |
| Body | font-body (Inter) | 14px (text-sm) | 500 | 1.5 | normal | Paragraph text, descriptions, detail rows |
| Label | font-mono (JetBrains) | 9px (text-[9px]) | 500 | 1.4 | 0.15em uppercase | ALL form labels, section subheadings, badge text |

Declared weights: 500 (body text, labels) and 700 (headings, display). Two weights only.

Notes:
- Mono labels use `tracking-[0.15em] uppercase` always — never deviate
- Body text weight is 500 (set globally in globals.css: `p { font-weight: 500 }`)
- Label weight is 500 — aligns with the global `p { font-weight: 500 }` project rule
- Secondary body text (table rows, descriptions): use `text-text-mid` not `text-text`

## Page Contracts

### Wizard Final Step — Review & Pay (Step 4 replacement)

Layout: `max-w-3xl mx-auto px-6 py-10` — identical to all other wizard steps.

Sections (top to bottom):
1. StepHeader — step number 5, title "Review & Pay", description "Your complaint is ready. Complete payment to file."
2. Filing summary card (`bg-bg-alt border border-border rounded-[6px] p-6`) — complaint type, target business, agency (CA AG only at launch)
3. Truthfulness attestation — `bg-bg-alt border border-border rounded-[6px] p-4` — mono label + body text attestation paragraph
4. Cost breakdown — `border border-bg-dark rounded-[6px] p-6 bg-bg-alt` — matches existing cost section pattern — shows "Privacy Complaint Filing" line item at $1.99
5. StepNavigation — `isLast={true}`, `continueLabel="Pay & File — $1.99 →"`, disabled while loading

Loading state: CTA button shows "Redirecting to Stripe..." and is disabled. No spinner — text change only.

Error state: `bg-accent-bg border border-accent rounded-[6px] p-4` directly above StepNavigation — mono label "Payment Error" + body text message.

### Success Page — /filing/[id]/success

Layout: `min-h-screen bg-bg` with Masthead + Footer. Content: `max-w-3xl mx-auto px-6 py-12`.

Sections (top to bottom):
1. Confirmation header — centered, `font-serif text-6xl text-border-dark` checkmark glyph (✓), then DoubleRule, then display heading "Complaint Filed"
2. Receipt ID card (`bg-bg-alt border border-border rounded-[6px] p-6`) — mono label "Filing Receipt" + large mono receipt ID (EFC-YYYYMMDD-XXXXX)
3. Filing detail rows — `bg-bg-alt border border-border rounded-[6px] p-6` — business name, agency, amount paid ($1.99), filed date; each row: `flex justify-between font-body text-sm text-text-mid`
4. PDF download section — if `complaintPdfUrl` is set: link button "Download Complaint PDF ↓"; if null: mono text "Your complaint PDF will be available shortly."
5. Account creation CTA (if `Filing.userId === null`) — `bg-bg-alt border border-border rounded-[6px] p-6` — heading "Track Your Filings", body + shadcn Button primary action "Create Free Account →"
6. Secondary action — centered link "File Another Complaint" — `font-mono text-[11px] tracking-[0.1em] uppercase border border-bg-dark text-text px-8 py-3 rounded-[6px]`

Payment status badge on success page: `font-mono text-[8px] tracking-[0.1em] uppercase px-2 py-0.5 rounded-[4px] bg-bg-dark text-white` — text "Paid"

## Accessibility

- All interactive elements reachable by keyboard (Tab order: Back → Continue)
- StepNavigation `disabled` attribute set on button element (not just CSS `cursor-not-allowed`)
- Account CTA button uses shadcn Button which has correct ARIA roles by default
- Success page checkmark glyph is decorative — wrap in `aria-hidden="true"`
- Receipt ID: wrap in `<output>` element or add `aria-label="Filing receipt ID: {id}"` for screen readers
- Minimum touch target on primary CTA: 44px height via `py-3.5` (exception noted in spacing scale)
- Color contrast: accent (#b91c1c) on white (#ffffff) = 5.9:1 — passes AA for normal text
- Color contrast: text (#1c1c1c) on bg-alt (#f5f5f3) = 17.5:1 — passes AAA
