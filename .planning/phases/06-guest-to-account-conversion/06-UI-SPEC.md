Layout: `max-w-3xl mx-auto px-6 py-12` (same as success page)

1. `Masthead` (sticky, unchanged)
2. `DoubleRule` divider

- Loading state: button shows spinner (lucide `Loader2` className `animate-spin`) + `disabled` attribute + text "Creating Account..."

## Interaction Contracts

### Form submission flow (account create)

1. Client-side validate on submit (empty fields, password length, password match)
2. If validation fails: show inline errors, do NOT submit
3. If validation passes: disable form, show loading state on button
4. POST to `/api/auth/register`
5. On 201: call `signIn('credentials', { email, password, redirect: false })`, then `router.push('/account/filings')`
6. On 409 (email taken): show email-taken error card, re-enable form
7. On 400/500: show generic error card, re-enable form

### Form submission flow (login)

1. Disable form on submit
2. Call `signIn('credentials', { email, password, redirect: false })`
3. On success: `router.push('/account/filings')`
4. On error: show invalid-credentials error card, re-enable form, clear password field only (retain email)

### PDF download (filing history)

- Link points to `/api/filings/[id]/pdf` (server-proxied)
- Opens in new tab (`target="_blank" rel="noopener noreferrer"`)
- No client-side download logic — server route verifies session and streams/redirects to Vercel Blob

### Middleware redirect

- Unauthenticated `/account/*` requests redirect to `/login`
- After successful login, redirect to `/account/filings` (not back to `/login`)

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Password input labeling | `<label>` associated via `htmlFor` — use `FormField` pattern |
| Read-only email field | `readOnly` + `aria-label="Email address (pre-filled from your filing)"` |
| Error messages | `role="alert"` on error cards so screen readers announce them immediately |
| Loading state | `aria-disabled="true"` + `aria-busy="true"` on button during submission |
| Filing status badge | `aria-label="Status: Filed"` on badge element |
| PDF link | `aria-label="Download complaint PDF for filing {filingReceiptId}"` |
| Form submission error | Focus moved to error card on appearance (`ref` + `focus()`) |
