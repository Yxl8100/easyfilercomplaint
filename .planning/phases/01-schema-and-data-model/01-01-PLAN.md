```prisma
enum FilingStatus {
  draft
  pending_payment
  paid
  generating
  filing
  filed
  failed
}
```

**Step 2: Modify Filing model** — add new fields and change existing ones:

Change `userId String` to `userId String?`
Change `user User @relation(fields: [userId], references: [id])` to `user User? @relation(fields: [userId], references: [id])`
Change `status String @default("draft")` to `status FilingStatus @default(draft)`

Add these new fields (all nullable, placed after `invoiceId`/`invoice` and before `status`):

```prisma
  // Stripe payment fields (SCHEMA-01)
  stripeSessionId   String?
  stripePaymentId   String?
  paymentStatus     String?
  paymentAmount     Decimal?  @db.Decimal(10, 2)
  paidAt            DateTime?

  // Phaxio fax fields (SCHEMA-02)
  faxId             String?
  faxStatus         String?
  faxSentAt         DateTime?
  faxCompletedAt    DateTime?
  faxPages          Int?

  // Filing receipt ID (SCHEMA-03)
  filingReceiptId   String?   @unique

  // Evidence file fields (SCHEMA-06)
  evidenceFileUrl   String?
  evidenceFileName  String?

  // PDF and email tracking (needed by Phase 3 and Phase 5)
  complaintPdfUrl   String?
  receiptEmailSentAt DateTime?
```

Note: The existing `amountPaid Decimal? @db.Decimal(10, 2)` field on Filing is the consumer's payment to the target business, NOT the $1.99 filing fee. The new `paymentAmount` is the Stripe filing fee. Both fields coexist.
